HTML

  • The head element contains information about the webpage (the metadata).
  • The body element represents the visible content shown to the user.
  • The meta element is information on the page that isn't visible but is used by the browser.
  • A semantic element means the name of the element describes its content and the role it plays.
  • Semantic elements make a page easier to read.
  • Div creates a division on the page.
  • Span creates an inline division.
  • An element is everything inside of the code from the opening tag to the closing tag.
  • The Section goes inside of the Body of HMTL code.
  • ID specifies the unique id for HTML.

CSS

  • There are 3 ways to style a webpage with CSS
  • Inline CSS, Internal CSS, External CSS
  • An External Style Sheet is the most common way to style a webpage with CSS.
  • Separation of Concerns: Each section of code should have its own responsibility.
  • CSS terminology includes: selector, property, and value.
  • A selector defines the element or attributes to which the rules or declarations will apply.
  • Declarations have 2 components: 1) the CSS property we want to apply 2) the value of that property.
  • An example of a property would be color.
  • An example of a value would be the blue.
  • There are margins, borders, padding, and content in HTML boxes.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Margins are not visible on the page.
  • An asterisk * in CSS denotes all of the properties on the page.
  • A class attribute allows one to share a CSS rule to any element chosen by assigning a rule to the class attribute with a class selector.

Git

  • git status: checks out what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git branch: lists your branches. a * will appear next to the currently active branch
  • git commit -m: commits staged content as a new commit snapshot
  • git stash: saves modified and staged changes
